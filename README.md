# CS-320

How can I ensure that my code, program, or software is functional and secure?

To ensure both functional and secure software it requires a proactive approach to quality assurance. Utilizing frameworks like JUnit allow for verification of individual code units. By designing test suits that target specific conditions developers can confidently say that the software will meet all functional requirements. Security software starts at the code level. Making sure parameters are validated through defensive coding and input validation prevents invalid data states and protects against injection attacks.

How do I interpret user needs and incorporate them into a program?

Interpreting user needs involves breaking down the needs of the stakeholder into structured constraints. Analyzing requirements carefully ensures that the implementation matches what the stakeholder is expecting. Having a test-driven perspective of requirements will help clarify the expected behavior of the code before any code it written. 

How do I approach designing software?

Approaching software design for has changed since this course. There needs to be a balance between scalability, maintainability, and security. Before writing any code, make sure everything is planned out and see how they interact. Having an iterative approach will allow for continuous feedback, regular testing, and incremental refactoring. This will ensure that software quality is built from the ground up.
